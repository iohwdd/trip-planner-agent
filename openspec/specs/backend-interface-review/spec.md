## 需求
### 需求:现有后端接口必须经过场景化审查并收敛契约
系统必须对现有聊天、会话、结果、计划和相关状态接口进行阶段性审查，明确其在成功、澄清、降级、失败和权限受限等场景下的返回契约。

#### 场景: 审查会话与回合接口
- **当** 团队对聊天会话与回合接口进行复核
- **那么** 必须明确每个接口在不同阶段下返回的关键状态字段、结果字段和错误语义

### 需求:接口错误语义必须可被前端直接消费
系统必须为关键接口提供可理解的错误文案和稳定的结构化错误信息，禁止仅依赖模糊字符串让前端推断错误类型。

#### 场景: 返回权限或状态冲突错误
- **当** 用户触发未登录访问、并发回合冲突或非法状态转换
- **那么** 接口必须返回稳定的 HTTP 状态码以及前端可直接消费的结构化错误信息

### 需求:流式接口必须与既有接口协同工作
系统必须保证新增流式接口与现有保存详情、查询详情、轮询回合和会话恢复接口可以协同工作，避免前端因为切换消费路径而丢失会话一致性。

#### 场景: 流式中断后恢复
- **当** 流式连接中途断开或客户端刷新页面
- **那么** 前端必须可以通过现有会话或回合详情接口恢复到一致的最新状态
