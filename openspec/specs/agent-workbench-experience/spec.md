## 需求
### 需求:主工作台必须突出旅游规划 Agent 定位
系统必须在主工作台中清晰表达“旅游规划 Agent”定位，并优先展示用户在当前阶段最关心的信息，而不是重复性的装饰性标题或泛化页面说明。

#### 场景: 初始进入工作台
- **当** 用户首次进入工作台或打开一个新会话
- **那么** 界面必须优先展示可立即开始对话的输入区域、关键引导和旅游规划相关的阶段提示

### 需求:聊天窗口顶部信息必须降噪
系统必须移除或隐藏聊天窗口上方不必要的重复标题信息，并将顶部操作按钮以低干扰方式呈现，禁止其在默认状态下压过主要对话内容。

#### 场景: 查看聊天区头部
- **当** 用户进入聊天工作台
- **那么** 聊天区顶部不得出现重复性的标题信息，且操作按钮必须采用弱化视觉权重或按需显示的方式

### 需求:不同阶段的界面结构必须收敛
系统必须在初始引导、执行中、澄清中、结果完成等阶段使用统一的布局逻辑和视觉语言，让用户能够快速识别当前所处阶段及下一步可执行动作。

#### 场景: 从执行中切换到结果完成
- **当** 一个回合从执行中进入结果完成状态
- **那么** 界面必须保留稳定的整体结构，只更新主要上下文内容和阶段提示，而不是让用户重新学习布局
