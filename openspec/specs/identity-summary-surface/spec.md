## 需求
### 需求:前端必须消费统一的身份摘要与资产摘要
系统必须以前端可直接消费的统一身份摘要为基础驱动关键页面，禁止完全依赖会话列表、方案列表等详细接口在前端二次拼装产品级状态。

#### 场景:应用初始化
- **当** 前端恢复当前身份状态
- **那么** 系统必须同时获得游客/登录状态、资产概览和关键能力开关，用于驱动壳层与关键页面

### 需求:关键页面必须展示与当前身份相关的能力边界
系统必须在工作台、资产页或身份入口中展示当前用户可以执行的关键动作边界，例如是否可保存方案、是否可管理历史方案、是否处于游客迁移前状态。

#### 场景:游客查看保存相关入口
- **当** 游客在工作台或资产页接近保存方案等受限动作
- **那么** 页面必须明确提示当前限制和登录后的能力变化

#### 场景:已登录用户查看身份摘要
- **当** 已登录用户查看壳层或身份页面
- **那么** 页面必须显示与当前账号相匹配的资产计数、最近会话或关键能力摘要

### 需求:前端必须用统一摘要减少重复加载与状态猜测
系统必须尽量用统一摘要接口驱动产品级提示，避免多个页面分别加载后再推断同一类产品状态，防止游客态与登录态下出现不一致展示。

#### 场景:身份切换后刷新界面
- **当** 用户从游客态切换到登录态，或从登录态退出
- **那么** 顶部提示、资产概览和关键操作可见性必须基于同一份最新身份摘要同步更新

### 需求:统一身份摘要必须暴露知识库管理能力
系统必须通过统一身份摘要向前端暴露知识库管理能力开关，使壳层导航和管理员页面可见性能够基于同一份身份状态驱动。

#### 场景:管理员恢复登录态
- **当** `is_staff=true` 的用户请求当前身份摘要
- **那么** 系统必须返回可直接消费的知识库管理能力标记
- **那么** 前端必须能够据此展示知识库导航入口和管理页面

#### 场景:普通用户恢复登录态
- **当** 未登录用户或 `is_staff=false` 的用户请求当前身份摘要
- **那么** 系统必须返回关闭的知识库管理能力标记
- **那么** 前端不得错误展示知识库入口
