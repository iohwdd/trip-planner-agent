## 需求
### 需求:工作台必须是流程化、表单式的规划入口
系统必须将旅行规划工作台定义为结构化表单驱动的流程入口，禁止要求用户先进入对话会话才能生成规划结果。

#### 场景:用户开始一次新规划
- **当** 用户进入流程工作台
- **那么** 页面必须优先展示结构化规划表单、运行状态和结果区域，而不是聊天消息流

### 需求:工作台必须使用独立的规划运行链路
系统必须通过独立的规划运行接口接收结构化输入并返回运行状态与结果，禁止将工作台提交隐藏为聊天消息。

#### 场景:用户提交工作台表单
- **当** 用户提交规划表单
- **那么** 前端必须调用规划运行接口，并根据运行状态更新执行过程与结果视图

### 需求:工作台结果必须可直接保存为方案资产
系统必须允许用户直接将结构化规划运行结果保存为旅行方案，禁止要求用户通过聊天会话间接保存。

#### 场景:工作台生成可用结果后
- **当** 当前规划运行已经返回可消费的规划结果
- **那么** 用户必须可以直接将该结果保存为草案或最终方案
