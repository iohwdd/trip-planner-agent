## 需求
### 需求: 助手消息必须支持流式输出
系统必须为独立助手会话提供流式消息接口，使前端能够在一次回复过程中持续接收消息增量与完成事件。

#### 场景: 发送一条助手消息
- **当** 客户端以流式模式提交助手消息
- **那么** 服务端必须返回可持续读取的事件流，并至少发送 `message.delta` 与 `message.complete` 事件

### 需求: 助手界面必须在输出时自动跟随
系统必须在助手回复增量输出期间自动将滚动区域保持在底部，且发送消息时不得出现明显布局抖动。

#### 场景: 助手流式回复进行中
- **当** 新的消息增量抵达
- **那么** 聊天滚动区域必须自动跟随到最新内容，并保持输入区位置稳定
