## ADDED Requirements

### 需求:输出必须支持路线规划结构
系统必须输出面向路线规划的结构化结果，而不仅是通用旅行摘要。

#### 场景:路线规划完成
- **当** 系统完成一次可用的路线规划
- **那么** 输出必须包含路线摘要、停靠点顺序、路段衔接、时间安排和预算信息
- **那么** 输出必须能够被前端直接分区渲染

### 需求:输出必须区分澄清态与最终态
系统必须在结果中明确区分“仍需澄清”和“可展示完整路线”两种状态。

#### 场景:关键信息仍然缺失
- **当** 用户尚未提供生成路线所需的关键信息
- **那么** 系统必须返回澄清问题和已确认约束
- **那么** 系统不得伪造完整路线结果

### 需求:输出必须包含替代方案与修订线索
系统必须在适用时提供可替代路线、调整建议或修订线索，以支持用户继续通过对话优化路线。

#### 场景:存在多个可行路线
- **当** 系统识别出两条及以上可行的路线组织方式
- **那么** 输出必须包含至少一个替代方案或路线调整建议
- **那么** 系统必须说明替代方案与主路线的主要差异
