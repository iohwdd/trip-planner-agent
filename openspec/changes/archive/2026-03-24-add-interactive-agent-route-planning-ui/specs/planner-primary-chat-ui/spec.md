## ADDED Requirements

### 需求:聊天工作台为主入口
系统首页必须默认展示对话式 Agent 工作台，聊天区必须是用户进入产品后最先看到和最易操作的规划入口。

#### 场景:用户首次打开首页
- **当** 用户进入应用首页
- **那么** 系统必须默认显示聊天输入区、会话记录区和当前规划状态
- **那么** 系统不得要求用户先填写完整表单才能开始规划

### 需求:表单入口降级为次级快捷方式
系统必须保留一键生成表单能力，但该入口必须作为聊天工作台中的次级入口存在，而不是主布局的第一层焦点。

#### 场景:用户改用表单快速填写
- **当** 用户选择使用快捷表单
- **那么** 系统必须在聊天主界面内打开表单面板、抽屉或折叠区
- **那么** 表单提交后必须进入同一套规划流程并更新主结果区

### 需求:聊天界面展示执行过程与结果联动
系统必须在聊天主界面中同步展示当前回合的执行步骤、数据源状态和路线结果摘要。

#### 场景:用户发送新的规划消息
- **当** 聊天回合进入执行状态
- **那么** 系统必须展示当前回合的执行步骤和数据源状态
- **那么** 在回合完成后，系统必须把新的路线结果显示在结果区，而不是要求用户跳转到单独页面
