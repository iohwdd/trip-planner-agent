## ADDED Requirements

### 需求:路线结果包含路段衔接信息
系统必须为已选定的路线站点生成点位之间的路段衔接信息，包括推荐交通方式和估算时长。

#### 场景:存在多个连续停靠点
- **当** 系统生成包含两个及以上停靠点的路线方案
- **那么** 结果中必须包含相邻停靠点之间的路段信息
- **那么** 每个路段必须至少包含起点、终点、推荐交通方式和估算时长

### 需求:路径规划优先基于地图能力
系统必须优先使用地图路径规划或路段估算能力生成交通衔接建议，而不是完全依赖模型自行编造路线细节。

#### 场景:地图路径能力可用
- **当** 地图路径规划接口返回有效结果
- **那么** 系统必须将该结果用于路段时长和交通方式建议
- **那么** 系统必须在来源信息中标注该路段数据的提供方

### 需求:路径能力失败时必须降级说明
当地图路径规划能力不可用时，系统必须返回降级的路段建议，并明确说明该路段为估算结果。

#### 场景:路径规划请求失败
- **当** 路径规划接口超时、失败或无可用结果
- **那么** 系统必须返回可读的启发式路段建议或替代说明
- **那么** 系统必须在结果中标记该路段为降级或估算数据
