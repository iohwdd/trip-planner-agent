## 新增需求

### 需求:主工作台必须围绕当前阶段和下一步动作组织
系统必须将主工作台组织为“当前阶段摘要 + 下一步动作 + 支撑信息”的决策流，禁止默认让聊天区、结果区和执行区以平级模块形式竞争用户注意力。

#### 场景:初始进入工作台
- **当** 用户打开一个新会话或空白工作台
- **那么** 界面必须优先展示如何开始规划、当前已知信息为空以及建议的首个动作

#### 场景:会话进入澄清态
- **当** 当前回合需要用户补充信息
- **那么** 工作台必须优先展示待补充项和推荐回答入口，并弱化与当前动作无关的内容

#### 场景:会话已有可消费结果
- **当** 当前回合已经生成路线草案或最终结果
- **那么** 工作台必须优先展示当前结论、可信度摘要和最有价值的继续修订动作

### 需求:工作台必须向用户解释当前结果是否可直接使用
系统必须在结果摘要层明确表达该版本是草案、最终态、澄清态还是降级态，并说明哪些内容仍需用户决策或确认。

#### 场景:显示降级结果
- **当** 系统返回带有降级信息的结果
- **那么** 工作台必须清晰标出受影响范围、对决策的影响和下一步建议

#### 场景:显示草案结果
- **当** 系统返回可继续细化的路线草案
- **那么** 工作台必须提示用户这版结果适合继续修订还是适合保存为草案

### 需求:工作台必须提供统一的下一步动作入口
系统必须提供统一的下一步动作层，用于承载继续提问、补充澄清、保存方案、跳转历史资产或继续修订等高价值动作，禁止把这些动作分散到难以发现的多个区域。

#### 场景:用户完成一轮规划后准备继续操作
- **当** 当前回合结束且系统已有明确状态
- **那么** 用户必须可以在一个稳定区域内看到最相关的后续动作，而不必自行搜索页面各角落
