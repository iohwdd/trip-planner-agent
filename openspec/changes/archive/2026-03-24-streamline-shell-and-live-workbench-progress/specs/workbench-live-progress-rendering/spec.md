## 新增需求

### 需求: 工作台初始态必须只展示表单
系统必须让工作台在尚未发起生成前仅展示结构化表单，禁止预先暴露结果区或过程说明区。

#### 场景: 首次进入工作台
- **当** 用户尚未点击生成旅行方案
- **那么** 页面只展示表单主区，不展示结果面板和执行时间线

### 需求: 工作台必须实时渲染规划进度
系统必须根据规划运行步骤的实时状态渲染进度区，让用户能看到当前步骤、已完成步骤和整体进展。

#### 场景: 运行中的工作台
- **当** 用户点击生成旅行方案且运行处于排队或执行中
- **那么** 页面必须显示当前步骤、整体进度和已完成阶段的实时状态

### 需求: 工作台完成态必须轻量提示
系统必须在规划完成后以轻量但显著的完成提示替代大段说明，并直接给出保存草案与保存最终方案动作。

#### 场景: 工作台完成生成
- **当** 规划运行进入完成状态
- **那么** 页面必须显示简洁完成提示，并提供保存草案与保存最终方案入口
