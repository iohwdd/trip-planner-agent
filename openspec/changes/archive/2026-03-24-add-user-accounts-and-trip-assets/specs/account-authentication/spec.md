## ADDED Requirements

### 需求:游客可以在未登录状态下开始规划
系统必须允许未登录用户以游客身份进入“旅游规划 Agent”工作台、创建临时会话并完成基础对话规划，而不得要求用户在首条消息前先登录。

#### 场景:游客首次进入工作台
- **当** 未登录用户打开首页并发送第一条旅行需求
- **那么** 系统必须创建一个可恢复的游客会话标识
- **那么** 系统必须允许用户继续聊天和查看结果

### 需求:用户必须能够通过邮箱验证码建立和结束登录态
系统必须支持用户通过邮箱验证码完成注册或登录，并在成功后返回可用的登录态；系统也必须支持用户主动退出登录。

#### 场景:用户首次使用邮箱登录
- **当** 用户提交一个尚未注册的邮箱并完成验证码校验
- **那么** 系统必须创建该用户账号
- **那么** 系统必须返回有效的登录凭证和当前用户信息

#### 场景:已存在用户再次登录
- **当** 已注册用户提交邮箱并通过验证码校验
- **那么** 系统必须登录到原有账号
- **那么** 系统不得重复创建新账号

#### 场景:用户主动退出
- **当** 已登录用户发起退出登录
- **那么** 系统必须使当前登录凭证失效
- **那么** 后续受保护接口请求必须被视为未登录

### 需求:系统必须支持查询当前登录状态
系统必须提供查询当前登录用户状态的能力，以便前端在刷新页面后恢复用户菜单和受保护数据。

#### 场景:前端刷新页面后查询用户信息
- **当** 客户端携带有效登录凭证请求当前用户信息
- **那么** 系统必须返回当前用户的标识、展示名称和登录状态
- **那么** 系统必须允许前端继续加载该用户的会话和方案资产

### 需求:游客资产必须在登录后迁移到账户下
系统必须允许游客在登录或注册成功后，把当前浏览器下的临时会话和相关方案资产迁移到新登录的用户账号。

#### 场景:游客在保存方案时触发登录
- **当** 游客在已有临时会话和结果的情况下完成登录
- **那么** 系统必须把当前游客会话归属迁移到该用户账号
- **那么** 用户登录后必须能够立即在“我的会话”或“我的方案”中看到该资产

### 需求:认证接口必须具备验证码有效期与频控
系统必须对验证码发送和校验接口实施有效期校验与频率限制，防止重复发送和暴力尝试。

#### 场景:用户短时间内重复请求验证码
- **当** 同一邮箱或同一请求来源在限制窗口内重复请求验证码
- **那么** 系统必须拒绝超限请求并返回明确的频控提示
- **那么** 系统不得继续发送新的验证码

#### 场景:用户提交过期验证码
- **当** 用户提交已过期或已使用的验证码
- **那么** 系统必须拒绝登录
- **那么** 系统必须返回明确的错误原因
